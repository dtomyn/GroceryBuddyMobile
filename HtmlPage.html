Now that the front end web site has been started, it is time to create a “working prototype”. That is, I wanted to put together a set of screens that looked similar to those drawn up in “Part 8” into a “semi-working” application.
For this, I used the knowledge I gained using training I took from Lynda.com, reading the docs at jQuery Mobile and playing with the online designer at jQuery Mobile as shown below to come up with what I liked for screens
Without getting into extensive training regarding jQuery mobile, I will give a brief overview of what I know and how I applied it to “Grocery Buddy”.
First of all, I took the *very* easy approach in that ALL of my HTML is actually on a single page and I did not use templating engines at all. That is, I attempted to keep things very straightforward.
Now that I have that out of the way, here are some high level jQuery basics and approaches that I used:
  • Each “page” is marked with an attribute of data-role="page". There are multiple pages in a single page application… I choose to use a “div” element for each of my pages but presumably I could have used an HTML5 article instead… perhaps one day I will take a closer look J 
  • A page then has attributes that you set:
-          id that will be referenced when navigating from page to page
-          data-theme that gives a page an overall theme / style
-          data-title that will display in the chrome of a web browser
  • Within each “page” there are the typical 3 areas… all attributed using a specific data-role
-          data-role="header" …meant to be at the top of the page (obviously). You can also set the following attribute which will ensure it will always display and will always be at the top data-position="fixed"
-          data-role="content" …as expected, this is where you would display all of your information for the page
-          data-role="footer" …similar to the previously mentioned "header" it is meant to be at the bottom of the page
  •   Within the header and footer you can then also put what result in UI navigation links… for example a “Home” and “Info” buttons
  •   While creating the UI navigation links you have a number of options of interest including:
-          data-icon …which you can use either your own custom icon or one of the built-in ones as described here
-          data-transition …which you can use to animate the display as pages are navigated to. You can find a listing of all of the options here
